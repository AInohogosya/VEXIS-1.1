# Contributing to VEXIS-1.1

Welcome to VEXIS-1.1!  
This project is still under development, but we strive to make it easy for anyone to join.  
Whether you write code, find bugs, share ideas, or improve documentation—all contributions are welcome!

## Types of Contributions (Especially welcome)

- **Bug fixes**  
  → If you find an issue with reproducible steps, please submit a PR!

- **Implementing new features or improvements**  
  → New OS operation use cases (e.g., enhanced Excel operations, browser automation stabilization)  
  → Adding other LLM providers (Anthropic Claude, OpenAI, etc.)  
  → Expanding memory functionality (improving long-term memory accuracy, adding editing features)

- **Performance and Stability Improvements**  
  → Accelerating screenshot processing  
  → Strengthening error recovery  
  → Improving compatibility on Windows/Linux

- **Adding/Enhancing Tests**  
  → We want to increase coverage with pytest (especially for vision/execution parts)

- **Improve Documentation & User Guide**  
  → Enhance USAGE.md, add troubleshooting  
  → Add examples with actual screenshots and videos

- **Ideas & Feedback**  
  → Submit discussions or issues like "I want this operation" or "This part is hard to use"

## Contribution Process (Step-by-Step)

1. **Let's Talk First**  
   - If you know what you want to do → Post "I want to do this" in GitHub Issues or Discussions  
   - For small fixes, you can submit a PR directly
